# p2-favorite-teams: お気に入りチーム

## 背景

ユーザーが応援チームを登録し、トップページや通知でそのチームの試合を
優先表示できるようにする。パーソナライゼーションの第一歩として、
実装コストが低い割にエンゲージメント向上効果が高い。

## スコープ

対象:
- お気に入りチームの登録・解除（最大 3 チーム）
- トップページでのお気に入りチーム試合の優先表示
- push 通知のチームフィルタへの連携（p2-push-notifications と統合）

対象外:
- チームページ（将来フェーズ）
- お気に入りチームに基づくコンテンツレコメンド

## データモデル変更

### `user_profiles` への追加カラム（p2-auth のテーブルを拡張）

| カラム | 型 | 説明 |
|---|---|---|
| favorite_team_slugs | text[] | お気に入りチームの slug 配列（最大 3 件）|

RLS: 本人のみ read/update 可。

## API サーフェス

| ルート | メソッド | 役割 |
|---|---|---|
| `PATCH /api/user/profile` | PATCH | `favorite_team_slugs` を更新 |

バリデーション:
- slug は `teams` テーブルに存在するもののみ
- 最大 3 件。超過した場合は 400 を返す

## UI サーフェス

### チーム選択 UI（`components/team-picker.tsx`）

- 対応チームをフラグ + チーム名で一覧表示
- 選択済みはハイライト、3件で他をグレーアウト
- プロフィールページに配置

### トップページの優先表示

- お気に入りチームの直近試合を上部にピン表示
- 未設定ユーザーには「応援チームを登録する」バナーを表示（1回のみ）

## 受け入れ条件

- [ ] お気に入りチームを最大 3 チーム登録・解除できる
- [ ] トップページでお気に入りチームの試合が上部に表示される
- [ ] 未設定ユーザーへの登録バナーが表示される
- [ ] `push_subscriptions.team_slugs` にお気に入りチームが反映される
- [ ] `pnpm tsc --noEmit` と `pnpm build` が通る
